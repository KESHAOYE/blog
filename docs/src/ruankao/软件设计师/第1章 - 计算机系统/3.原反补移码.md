# 3、原反补移码

采用补码简化计算机运算部件的设计

## 原码

- 最高位为符号位，0 为正，1 为负

### 取值范围

-（2<sup>n-1</sup> -1） ～ （2<sup>n-1</sup> -1）

## 反码

- 正数和原码相同
- 最高位为符号位，0 为正，1 为负
- 负数就是除符号位，其余各位取反。

### 取值范围

-（2<sup>n-1</sup> -1） ～ （2<sup>n-1</sup> -1）

## 补码

- 正数和原码相同
- 最高位为符号位，0 为正，1 为负
- 负数为反码加一
- 正负 0 补码一样，都是 00000000
- 求补码的补码可以得到原码

### 取值范围

-（2<sup>n-1</sup> ） ～ （2<sup>n-1</sup> -1）

**因为正负 0 的补码一样，所以相当于没有-0，所以就有-128**

## 移码

- 在补码的情况下，符号位取反
- 正负 0 移码一样，都是 10000000

### 取值范围

-（2<sup>n-1</sup> ） ～ （2<sup>n-1</sup> -1）

**因为正负 0 的补码一样，所以相当于没有-0，所以就有-128**
